# SmartRepo AI 🔍

> **Ask questions about any GitHub repository in plain English.**

SmartRepo is a RAG (Retrieval-Augmented Generation) based question-answering system that indexes software repositories and lets developers query codebases using natural language. Point it at any GitHub repo and ask things like *"Where is authentication handled?"* or *"What does the payment service do?"* — and get precise, context-grounded answers.

---

## ✨ Features

- **Natural language queries** over any codebase
- **Hybrid search** — 60% ChromaDB semantic + 40% BM25 keyword retrieval
- **Context-aware answers** generated by a HuggingFace LLM
- **Multi-language support** — works on Python, JS, Solidity, and more
- **Lazy initialization** for fast cold starts on cloud deployment
- **REST API** via FastAPI for easy frontend integration

## 🏗️ Architecture

```
User Query
    │
    ▼
┌─────────────┐     ┌──────────────────────────────┐
│  Frontend   │────▶│         FastAPI Backend        │
│  (HTML/JS)  │     └──────────────┬───────────────┘
└─────────────┘                    │
                                   ▼
                    ┌──────────────────────────────┐
                    │        Hybrid Retriever        │
                    │  60% ChromaDB + 40% BM25       │
                    └──────┬───────────────┬────────┘
                           │               │
                    ┌──────▼──────┐ ┌──────▼──────┐
                    │  ChromaDB   │ │  BM25 Index  │
                    │  (Semantic) │ │  (Keyword)   │
                    └─────────────┘ └─────────────┘
                                   │
                                   ▼
                    ┌──────────────────────────────┐
                    │   HuggingFace Inference API   │
                    │   (Answer Generation)         │
                    └──────────────────────────────┘
```

**Indexing Pipeline:**
1. Clone target GitHub repository
2. Walk file tree and split into overlapping chunks
3. Embed chunks via `all-MiniLM-L6-v2` → store in ChromaDB
4. Build BM25 index over raw chunk text

**Query Pipeline:**
1. Embed user question
2. ChromaDB cosine similarity search (top-k)
3. BM25 keyword search (top-k)
4. Merge results with 60/40 weighting
5. Pass top-3 chunks as context to LLM → return answer

## 🚀 Getting Started

### Prerequisites
- Python 3.13+
- HuggingFace API key

### Installation

```bash
# Clone the repo
git clone https://github.com/AbhijitPatill/SmartRepo.git
cd SmartRepo

# Create virtual environment
python -m venv venv
venv\Scripts\activate        # Windows
# source venv/bin/activate   # Mac/Linux

# Install dependencies
pip install -r requirements.txt
```

### Configuration

Create a `.env` file in the root:
```env
HF_API_KEY=your_huggingface_api_key_here
```

### Run

```bash
# Start the backend
uvicorn backend.main:app --host 0.0.0.0 --port 8000

# Open frontend/index.html in your browser
```

## 🔌 API Reference

### Index a Repository
```http
POST /index
Content-Type: application/json

{
  "repo_url": "https://github.com/owner/repository"
}
```

### Query the Repository
```http
POST /query
Content-Type: application/json

{
  "question": "Where is user authentication handled?"
}
```

**Response:**
```json
{
  "answer": "Authentication is handled in backend/auth.py ...",
  "sources": ["backend/auth.py", "backend/middleware.py"]
}
```

## ☁️ Deployment

Deployed on **Render**. Key deployment considerations:

- **Lazy initialization** in `embedder.py` and `retriever.py` — models load on first request, not at startup (prevents Render timeout)
- **CPU-only torch** — pinned in `requirements.txt` to avoid GPU build conflicts
- **No `--reload` flag** in production uvicorn command
- **HF API key** stored as environment variable in Render dashboard — never hardcoded
