UNDER THE HOOD

How SmartRepo Works

A five-stage pipeline that converts raw source code into a semantic knowledge base, powered by RAG.

01
Input
Upload Repository
Accepts a zipped project or a Git URL. SmartRepo recursively walks the directory tree, filtering out binaries and irrelevant files to extract only meaningful source.
# Supported input methods
repo_path = "./my-project.zip"
repo_url = "https://github.com/user/repo"
load_repository(repo_path, extensions=[".py", ".js", ".md"])
02
Parsing
Parse & Chunk
Each file is parsed into logical units — functions, classes, and README sections. Chunks are kept within a token limit to fit cleanly into the LLM context window.
def chunk_file(content, max_tokens=512):
  chunks = split_by_function(content)
  return [c for c in chunks if token_count(c) <= max_tokens]
03
Embeddings
Generate Embeddings
Each chunk is passed through a SentenceTransformer model to produce a dense vector that captures semantic meaning — not just keywords.
from sentence_transformers import SentenceTransformer
model = SentenceTransformer("all-MiniLM-L6-v2")
embeddings = model.encode(chunks, batch_size=32)
04
Retrieval
Semantic Search via FAISS
Embeddings are stored in a FAISS index. On each query, the top-k most semantically similar chunks are retrieved in milliseconds.
import faiss
index = faiss.IndexFlatL2(embedding_dim)
index.add(embeddings)
_, top_k = index.search(embed(query), k=5)
05
Generation
LLM Generates Grounded Answer
Retrieved chunks are injected into a structured prompt and sent to a local Ollama LLM. The model generates an explanation grounded in actual code — no hallucinations.
prompt = build_prompt(query, retrieved_chunks)
response = ollama_generate(model="mistral", prompt=prompt)
return response["answer"]
